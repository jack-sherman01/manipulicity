# 基于视觉语言模型的机器人抓取物体质量估计方法

## 1. 方法概述

本方法利用视觉语言模型（Vision-Language Model，VLM）从单张 RGB 图像中估计机器人末端执行器抓取或悬吊物体的质量。核心思想是将物理推理任务形式化为一个结构化的视觉问答问题，借助 VLM 的多模态理解能力，模拟专家的逐步推理过程，最终输出质量估计值及置信区间。

---

## 2. 整体流程

```
输入图像（文件路径 / PIL.Image / np.ndarray）
        │
        ▼
  图像预处理：统一编码为 Base64 PNG
        │
        ▼
  VLM 推理（CoT 提示 + 结构化输出）
        │
        ▼
  响应解析：从混合文本中提取 JSON
        │
        ▼
  输出结果字典（质量估计 + 置信区间 + 材料推断等）
```

---

## 3. 图像预处理

**函数：`_to_base64_png`**

支持三种输入格式：
- 文件路径（`str` / `os.PathLike`）
- NumPy 数组（`uint8` 或 `[0,1]` 浮点，`H×W×C`）
- PIL `Image` 对象

所有输入统一转换为 **RGB PNG 格式**后进行 Base64 编码，作为 API 请求的内嵌图像数据（`data:image/png;base64,...`），保证跨平台和跨模型的格式一致性。

---

## 4. 提示词工程（Prompt Engineering）

### 4.1 系统提示（System Prompt）

系统提示将模型定位为"机器人感知领域的专家"，并将推理过程分为两个明确阶段：

#### 第一阶段：链式思维推理（Chain-of-Thought）

要求模型在提交估计数值之前，按照以下六个步骤进行自由散文式推理：

| 步骤 | 内容 |
|------|------|
| 1. 物体识别与描述 | 物体形状、尺寸（相对于夹爪/末端执行器）、外观特征 |
| 2. 材质推断 | 根据颜色、纹理、光泽和上下文推断材质（金属、塑料、木材、食物、织物、玻璃等） |
| 3. 尺寸估计 | 以机器人硬件为比例参考，估算长×宽×高或直径×高，需明确说明比例假设 |
| 4. 质量计算 | 利用估算体积和材质密度计算近似质量，展示计算过程 |
| 5. 合理性验证 | 检查结果是否符合该类型物体的物理常识，必要时调整并解释 |
| 6. 不确定性分析 | 说明估计的不确定性来源，给出质量下界和上界 |

这种显式的逐步推理（CoT）设计能够显著提升模型对复杂物理量估算的准确性，避免模型直接跳至结论。

#### 第二阶段：结构化输出

在自由推理之后，要求模型输出标准 JSON 块（使用 ` ```json ` 围栏），包含以下字段：

```json
{
  "mass_kg": <float>,               // 最佳估计质量（千克）
  "mass_kg_range": [lower, upper],  // 合理质量范围
  "material_guess": "<string>",     // 推断的材质
  "object_description": "<string>", // 物体简要描述
  "confidence": "<low|medium|high>",// 置信度
  "reasoning": "<string>"           // 推理过程的一句话摘要
}
```

### 4.2 用户提示（User Prompt）

用户提示简洁明了，仅指定任务场景：

> "请估计图像中物体的质量，该物体正被机器人末端执行器抓取或悬挂。"

## 5. VLM 后端支持

系统支持三种 VLM 后端，通过模型名称**自动检测**推理路由：

| 后端 | 检测规则 | 代表模型 | API 密钥来源 |
|------|----------|----------|--------------|
| **OpenAI** | 模型名不含特殊前缀 | `gpt-4o`、`gpt-4o-mini` | `api_key` 参数 / `OPENAI_API_KEY` 环境变量 |
| **Google Gemini** | 模型名以 `gemini` 开头 | `gemini-1.5-flash`、`gemini-1.5-pro` | `gemini_api_key` 参数 / `GEMINI_API_KEY` 环境变量 |
| **本地 VLM（Ollama）** | 模型名以 `ollama/` 开头，或设置了 `local_base_url` / `LOCAL_VLM_BASE_URL` | `llama3.2-vision:11b`（默认） | 无需密钥 |

### 5.1 OpenAI 后端

使用 OpenAI Chat Completions API，图像以 `image_url`（内嵌 Base64）+ `detail: high` 方式传入，确保高分辨率图像理解。

### 5.2 Gemini 后端

使用 `google-generativeai` SDK，通过 `GenerativeModel` 接口将系统提示作为 `system_instruction` 传入，图像以 PIL `Image` 对象直接传递给 `generate_content`。

### 5.3 本地 VLM 后端（Ollama）

复用 OpenAI SDK，通过自定义 `base_url`（默认 `http://localhost:11434/v1`）连接本地 Ollama 服务器，实现零成本的本地推理。默认模型为 `llama3.2-vision:11b`（约 8 GB 显存需求，性能与成本均衡）。`ollama/` 前缀在请求时自动剥离。

所有后端统一设置 `temperature=0.0`（确定性输出）和 `max_tokens=2048`（为 CoT 推理预留充足空间）。

## 6. 响应解析

**方法：`_parse_response`**

由于 CoT 设计使模型输出包含自由文本 + JSON 混合内容，解析逻辑采用两阶段策略：

1. **优先提取围栏 JSON**：搜索 ` ```json ... ``` ` 代码块，提取其中内容
2. **回退提取裸 JSON**：若无围栏，则从全文中提取最后一个 `{...}` 对象

提取后进行以下验证：
- JSON 格式合法性检查（`json.JSONDecodeError`）
- 必要字段完整性检查（缺失字段时抛出 `ValueError`）
- 数值类型强制转换（`mass_kg` → `float`，`mass_kg_range` → `list[float]`）

---

## 7. 输出格式

`estimate()` 方法返回一个 Python 字典，包含：

| 字段 | 类型 | 含义 |
|------|------|------|
| `mass_kg` | `float` | 最佳估计质量（千克） |
| `mass_kg_range` | `list[float, float]` | 质量合理范围 `[下界, 上界]`（千克） |
| `material_guess` | `str` | 推断的物体材质 |
| `object_description` | `str` | 物体简要描述 |
| `confidence` | `str` | 置信度：`"low"` / `"medium"` / `"high"` |
| `reasoning` | `str` | 推理摘要（一句话） |

---

## 8. 使用方式

### Python API

```python
from mass_estimator import MassEstimator

# OpenAI
estimator = MassEstimator(api_key="sk-...", model="gpt-4o-mini")

# Gemini
estimator = MassEstimator(gemini_api_key="AIza...", model="gemini-1.5-flash")

# 本地 Ollama
estimator = MassEstimator(model="ollama/llama3.2-vision:11b")

result = estimator.estimate("path/to/image.jpg")
print(result["mass_kg"], result["confidence"], result["reasoning"])
```

### 命令行

```bash
# OpenAI
python mass_estimator.py --image img.jpg --model gpt-4o-mini --api_key sk-...

# Gemini
python mass_estimator.py --image img.jpg --model gemini-1.5-flash --gemini_api_key AIza...

# 本地 Ollama
python mass_estimator.py --image img.jpg --model ollama/llama3.2-vision:11b
```
